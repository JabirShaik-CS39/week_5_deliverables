Forms & Validation in React — Deep Explanation

Forms are one of the most important parts of web applications because they collect user input. In modern React applications, developers commonly use:

React Hook Form (RHF) → Form state management
Zod → Validation
React Dropzone → File uploads
FastAPI → Backend API handling form data
1. React Hook Form (RHF)

React Hook Form is a lightweight library that manages form state efficiently with minimal re-renders.

Install:

npm install react-hook-form
Basic Example
import { useForm } from "react-hook-form";

function LoginForm() {
  const {
    register,
    handleSubmit,
    formState: { errors }
  } = useForm();

  const onSubmit = (data) => {
    console.log(data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input
        {...register("email", {
          required: "Email is required"
        })}
      />

      {errors.email && (
        <p>{errors.email.message}</p>
      )}

      <button type="submit">
        Login
      </button>
    </form>
  );
}
2. register()

register() connects an input field to React Hook Form.

Syntax
register("fieldName")

Example:

<input {...register("username")} />

RHF now tracks:

value
touched state
validation state
errors
Validation with register
<input
  {...register("password", {
    required: "Password required",
    minLength: {
      value: 6,
      message: "Minimum 6 characters"
    }
  })}
/>

Validation rules:

required
minLength
maxLength
pattern
validate

Example:

pattern: {
  value: /^\S+@\S+$/i,
  message: "Invalid email"
}
3. handleSubmit()

Handles form submission and validation.

const onSubmit = (data) => {
  console.log(data);
};

<form onSubmit={handleSubmit(onSubmit)}>

Process:

User clicks submit
        ↓
Validation runs
        ↓
No errors?
        ↓
onSubmit(data)

If validation fails:

onSubmit()

will NOT execute.

5. watch()

Observes field values in real time.

Example:

const password = watch("password");

Useful for:

Confirm Password
const password = watch("password");

<input
  {...register("confirmPassword", {
    validate: value =>
      value === password ||
      "Passwords do not match"
  })}
/>
Watch Entire Form
const values = watch();

console.log(values);

Output:

{
  email: "abc@gmail.com",
  password: "123456"
}
6. setValue()

Programmatically updates form fields.

Example:

setValue("username", "Jabir");

Automatically updates UI.

Common Use Cases
API Prefill
useEffect(() => {
  fetchUser().then(user => {
    setValue("name", user.name);
  });
}, []);
Edit Form
setValue("title", product.title);
7. Zod Validation

Zod is a TypeScript-first validation library.

Install:

npm install zod
Why Zod?

Without Zod:

required
minLength
pattern

spread everywhere.

With Zod:

Validation stays in one schema.

8. zodResolver

Install:

npm install @hookform/resolvers

Connects:

React Hook Form
       +
      Zod
Create Schema
import { z } from "zod";

const loginSchema = z.object({
  email: z
    .string()
    .email("Invalid email"),

  password: z
    .string()
    .min(6, "Minimum 6 characters")
});
Connect to RHF
const {
  register,
  handleSubmit,
  formState: { errors }
} = useForm({
  resolver: zodResolver(loginSchema)
});
Form
<input {...register("email")} />
<p>{errors.email?.message}</p>

<input {...register("password")} />
<p>{errors.password?.message}</p>

10. Register Form Pattern

More validation.

Fields:

name
email
password
confirmPassword

Schema:

const schema = z
  .object({
    password: z.string().min(6),
    confirmPassword: z.string()
  })
  .refine(
    data =>
      data.password === data.confirmPassword,
    {
      message: "Passwords must match",
      path: ["confirmPassword"]
    }
  );

12. Dynamic Fields (useFieldArray)

Used when users can add/remove inputs dynamically.

Install already included in RHF.

Example:

skills

User can add:

React
Node
FastAPI
Docker
Setup
const {
  control,
  register
} = useForm({
  defaultValues: {
    skills: [{ name: "" }]
  }
});

const {
  fields,
  append,
  remove
} = useFieldArray({
  control,
  name: "skills"
});
Render
{
  fields.map((field, index) => (
    <div key={field.id}>
      <input
        {...register(
          `skills.${index}.name`
        )}
      />

      <button
        type="button"
        onClick={() => remove(index)}
      >
        Delete
      </button>
    </div>
  ));
}
Add New Skill
<button
  type="button"
  onClick={() =>
    append({ name: "" })
  }
>
  Add Skill
</button>
13. Multi-Step Form (Wizard Pattern)

Large forms are divided into steps.

Example:

State

15. Send multipart/form-data to FastAPI

Files cannot be sent as JSON.

Instead:

multipart/form-data
Create FormData
const formData = new FormData();

formData.append(
  "title",
  data.title
);

formData.append(
  "image",
  data.image
);
Send
await axios.post(
  "/products",
  formData,
  {
    headers: {
      "Content-Type":
        "multipart/form-data"
    }
  }
);
FastAPI Backend
from fastapi import UploadFile, File

@app.post("/products")
async def create_product(
    title: str = Form(...),
    image: UploadFile = File(...)
):
    return {
        "filename": image.filename
    }
UploadFile Advantages
Streams file
Memory efficient
Supports large uploads

Methods:

await image.read()
await image.write()
image.filename
image.content_type
16. Form Accessibility

Accessibility ensures forms work for:

Screen readers
Keyboard users
Disabled users
Labels

Bad:

<input placeholder="Email" />

Good:

<label htmlFor="email">
  Email
</label>

<input
  id="email"
  type="email"
/>

Screen readers now know what field is being edited.

aria-invalid
<input
  aria-invalid={
    errors.email
      ? "true"
      : "false"
  }
/>

Indicates validation failure.

aria-describedby
<input
  aria-describedby="email-error"
/>

<p id="email-error">
  Email required
</p>

Screen readers automatically read the error.

Error Announcements
<p role="alert">
  Email required
</p>

role="alert" immediately announces errors to assistive technologies.

Keyboard Accessibility

Buttons should be:

<button type="submit">
  Save
</button>

Avoid:

<div onClick={save}>
  Save
</div>

because <div> is not keyboard accessible.
